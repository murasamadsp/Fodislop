NetworkService & ConnectionManager

Сетевое взаимодействие осуществляется по протоколу TCP через пакеты пакета MinesServer.Networking:

  • ConnectionManager — отвечает за установление TCP-соединения, отправку авторизационного пакета LoginRequestPacket и обработку обрывов связи.
  • NetworkService — управляющий синглтон подписки и отписки обработчиков входящих пакетов: 
    NetworkService.Instance.Subscribe<MapRegionPacket>(OnMapRegionReceived);
NetworkService.Instance.Unsubscribe<MapRegionPacket>(OnMapRegionReceived);
  • PacketHandler — центральный диспетчер. Принимает сетевые пакеты из очереди и делегирует их соответствующим менеджерам (MapManager, RobotManager, PackManager, SFXEffectManager, PacketUIBuilderFactory).

Основные типы входящих пакетов

Пакет Менеджер-обработчик Назначение
WorldInitPacket MapManager Первичная инициализация мира: размеры карты (Width/Height), конфигурация ячеек.
MapRegionPacket MapManager / MapStorage Обновление прямоугольного региона блоков на карте с RLE-декодированием.
RobotSpawnPacket / RobotMovePacket RobotManager Спавн и дельта-перемещение NPC/игроков в мировых coordinates Top-Left.
SFXPacket SFXEffectManager / ServerAudioEventManager Воспроизведение 3D-звука в FMOD + спавн визуального эффекта (Effekseer/GIF).
OpenWindowPacket PacketUIBuilderFactory Динамическая сборка окна UI Toolkit из дерева серверизованных компонентов.

Централизованная блокировка ввода игрока

Метод PacketHandler.IsInputBlocked объединяет состояние систем и блокирует ввод движения/копки в PlayerMovementController и PlayerInteractionController:

  • При открытом меню паузы (PauseMenu.IsMenuOpen == true).
  • При открытом инвентаре или окнах с полем ввода чата (ChatInput.IsFocused == true).
  • При открытых модальных серверизованных GUI-окнах.
