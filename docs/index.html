Архитектурный принцип клиенто-серверного взаимодействия

Клиент и сервер Fodinae работают по принципу четкого разделения тяжелой визуализации и легкого сетевого состояния:

  • Примитивы рендера на клиенте: В клиенте зафиксированы исполнительные компоненты отображения (тайлы мира, сущности роботов, предметов на земле, визуальных и звуковых эффектов).
  • Легковесный сетевой поток: Сервер передает клиенту только чистые координаты и идентификаторы состояний (где находятся роботы, где стоят паки предметов, какие ячейки попадают в радиус видимости, какие звуки вызваны вокруг).
  • Однократный запрос ассетов (On-Demand): Когда на экране впервые появляется неизвестный клиенту объект (новый тип блока, скин робота, иконка пака, аудио-банк), клиент запрашивает тяжелые ассеты (текстуры, спрайты, FMOD `.bank`) с сервера/CDN ровно один раз.
  • Автономный рендер из кэша: Все полученные ассеты сохраняются в дисковом кэше PersistentAssetCache (с ETag/MD5 валидацией) и ОЗУ. При последующих появлениях аналогичных объектов клиент строит тяжелый рендеринг строго из локального кэша, не обращаясь повторно к серверу.

Технологический стек

Клиент Fodinae построен на базe Unity 6 (6000.2.10f1) с использованием 2D Universal Render Pipeline (URP). Основные системные зависимости:

  • Рендеринг: 2D Render Pipeline (`com.unity.render-pipelines.universal` 17.2.0), процедурный меш SingleMeshTerrainRenderer.
  • Аудио: FMOD Studio C++ Engine via Native Wrapper, диск-стриминг `.bank` через loadBankFile.
  • Сеть: TCP сокеты через сетевой стек MinesServer.Networking (пакеты darkar25.fodinae.*).
  • Асинхронность: UniTask (Cysharp) без выделений GC на генераторах сопрограмм.
  • Интерфейс: UI Toolkit (`UIDocument`, UXML, TSS темы) + пакетная сборка окон PacketUIBuilderFactory.

Система координат и позиционирования

В проекте строго соблюдается единая система пространственных координат:

  • Серверные координаты: Точка `(0, 0)` находится в левом верхнем углу (поверхность мира). Ось X растёт вправо, ось Y растёт вниз (вглубь шахты).
  • Конвертация: Все пространственные преобразования централизованы в CoordinateUtils.
  • Единственный источник истины: Позиция игрока хранится в свойстве PlayerMovementController.Position (`Vector2Int` серверных координат).
