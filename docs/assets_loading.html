Концепция: Однократный запрос ресурса & Локальный рендер

Клиент Fodinae следует принципу ленивой однократной загрузки: клиент инкапсулирует исполнительные примитивы рендеринга (тайлы, сущности роботов, предмета паков, FMOD-звуки), в то время как сервер передаёт исключительно легковесные координаты и ID состояний.

При возникновении неизвестной клиенту сущности ресурс (текстура, спрайт, банк звуков) запрашивается с сервера/CDN ровно один раз. Все последующие появления того же объекта обрабатываются локальным кэшем без дублирования сетевых запросов.

Пайплайн загрузки ресурсов (ClientAssetLoader)

Загрузка ассетов с сервера (текстуры, аватары, анимированные спрайты, FMOD банки) выполняется через ClientAssetLoader:

  1. Проверка RAM-кэша (AssetCache): Быстрый поиск уже декодированного ресурса в памяти по URL/ключу.
  2. Дисковый кэш (PersistentAssetCache): Если ресурс отсутствует в RAM, проверяется локальный файл в persistentDataPath.
  3. HTTP ETag/MD5 Валидация: Если файл есть на диске, отправляется HTTP HEAD/GET запрос с заголовком If-None-Match: "ETag".
    • При ответе 304 Not Modified ресурс читается напрямую с диска без повторной скачки.
    • При ответе 200 OK скачивается новый поток байт и обновляется дисковый кэш.
  4. Дедупликация запросов (Coalescing): При одновременном вызове нескольких загрузок одного URL создаётся единый TaskCompletionSource, исключающий параллельные duplicate HTTP-запросы.
