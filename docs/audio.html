Архитектурный обзор аудио-домена

Аудиоподсистема Fodinae спроектирована как многоуровневая система, связывающая сетевые пакеты сервера, климатические зоны мира, UI микшер и низкоуровневый движок FMOD Studio C++.

  • Сетевой поток звуков (ServerAudioEventManager) — приём пакетов SFXPacket от сервера, расчёт затухания по дистанции, синхронизация с визуальными эффектами VFXPool.
  • Динамический фоновый звукоряд (WorldAudioController) — интеграция глубины шахты (параметр Depth), смены биомов, эмбиентных треков и плавных кроссфейдов.
  • Пространственное 3D-позиционирование (AudioSpatial & AudioZone) — нативное связывание движущихся объектов Transform с аудио-источниками и акустическими зонами FMOD Snapshots (пещеры, коридоры).
  • Микшер и пользовательские настройки (AudioBusType & PauseMenu) — централизованная регулировка 6 шин громкости с мгновенным сохранением в PlayerPrefs.
  • Низкоуровневый C++ бэкенд (AudioSystem & FmodAudioBackend) — ленивая загрузка аудио-банков .bank с диска через loadBankFile, нулевые GC-аллокации на воспроизведение.

1. Сетевые звуковые события (ServerAudioEventManager)

Сервер передаёт звуки мира через пакеты SFXPacket. Класс ServerAudioEventManager выступает связующим звеном:

  • Декодирует координаты Vector2Int и идентификатор звукового события.
  • Вызывает воспроизведение через AudioSystem.Instance.PlayAt(eventId, position).
  • Параллельно запрашивает спавн визуального эффекта (Effekseer / GIF-спрайт) из пула VFXPool.PooledSlot.

2. Фоновый звукоряд и динамика мира (WorldAudioController)

Класс WorldAudioController управляет атмосферой окружения по мере погружения игрока в шахту:

  • Глобальный параметр глубины: Транслирует текущую Y-координату игрока в FMOD через AudioSystem.Instance.SetGlobalParameter("Depth", currentDepth), меняя фильтрацию и реверберацию.
  • Переключение биомов: Активирует FMOD Snapshots (например snapshot:/Cave_Ambient или snapshot:/Lava_Zone) в зависимости от текущего региона карты.

3. Пространственное позиционирование (AudioSpatial & AudioZone)

Компонент Файл Функционал
AudioSpatial AudioSpatial.cs Компонент на GameObject. Вызывает FmodAudioBackend.AttachInstanceToGameObject для нативного C++ трекинга 3D-позиции без поллинга в C# Update.
AudioZone AudioZone.cs Триггерный коллайдер. При входе игрока активирует акустический FMOD Snapshot и выставляет локальные параметры эмбиента.

4. Управление микшером и шинами (AudioBusType)

Управление громкостью распределено по 6 независимым шинам, настраиваемым в PauseMenu:

AudioBusType Путь шины в FMOD Назначение PlayerPrefs Ключ
Master bus:/ Общий выходной микс Audio_Master
SFX bus:/sfx Звуки копки, взрывов, механизмов Audio_SFX
Music bus:/music Музыкальное сопровождение Audio_Music
Voice bus:/voice Озвучка персонажей и сообщений Audio_Voice
Ambience bus:/ambience Фоновые шумы пещер, эхо, лава Audio_Ambience
UI bus:/ui Звуки интерфейса, клики, модалки Audio_UI

5. Программирование вызовов (AudioSystem C# API)

// 2D UI звук
AudioSystem.Instance.Play2D("ui_click");

// 3D звук с нативной привязкой к роботу
AudioSystem.Instance.PlayAttached("robot_engine", gameObject);

// Акустический FMOD Snapshot
var snapshot = AudioSystem.Instance.PlaySnapshot("snapshot:/Cave_Ambient");

// Глобальный параметр FMOD (глубина шахты)
AudioSystem.Instance.SetGlobalParameter("Depth", 350f);

// Установка громкости шины SFX
AudioSystem.Instance.SetBusVolume(AudioBusType.SFX, 0.8f);
