Архитектура аудио-домена

Аудиоподсистема взаимодействует напрямую с **FMOD Studio C++ Engine**. Все звуковые события адресуются по именам событий (`event:/...` или `snapshot:/...`).

  • AudioSystem — глобальный синглтон-контроллер в DontDestroyOnLoad. Принимает вызовы Play, PlayAttached, PlaySnapshot, SetGlobalParameter, SetBusVolume.
  • FmodAudioBackend — низкоуровневая обёртка над C++ FMOD API (загрузка банков с диска через loadBankFile, нативная привязка к GameObject, управление сэмплами и шинами).
  • AudioPlaybackHandle — легкий хендл активного голоса, оборачивающий FMOD.Studio.EventInstance без C#-делегатов и аллокаций в памяти.
  • AudioSpatial — привязывает пространственные 3D-звуки к `Transform` с нативной трансляцией координат на C++ стороне FMOD.
  • AudioZone — триггерный регион, активирующий **FMOD Snapshots** (например snapshot:/Cave_Ambient) без перезаписи пользовательских настроек микшера.

Интерфейс программирования (C# API)

// 2D звук интерфейса
AudioSystem.Instance.Play2D("ui_click");

// 3D звук с нативной привязкой к GameObject
AudioSystem.Instance.PlayAttached("robot_engine", gameObject);

// Запуск FMOD Snapshot (акустика зоны)
var snapshot = AudioSystem.Instance.PlaySnapshot("snapshot:/Cave_Ambient");

// Глобальный параметр FMOD (глубина шахты)
AudioSystem.Instance.SetGlobalParameter("Depth", 350f);

// Установка громкости шины SFX
AudioSystem.Instance.SetBusVolume(AudioBusType.SFX, 0.8f);

Шины FMOD Studio (AudioBusType)

В микшере FMOD Studio зафиксированы 6 шин управления громкостью:

AudioBusType Путь шины в FMOD Назначение PlayerPrefs Ключ
Master bus:/ Главный выходной канал проекта Audio_Master
SFX bus:/sfx Игровые эффекты (копка, шаги, взрывы) Audio_SFX
Music bus:/music Фоновые музыкальные треки Audio_Music
Voice bus:/voice Голоса персонажей и диктора Audio_Voice
Ambience bus:/ambience Эмбиент окружения (пещеры, эхо, ветер) Audio_Ambience
UI bus:/ui Интерфейс, клики, открываемые окна Audio_UI

Обработка сетевых SFX

В ServerAudioEventManager при получении SFXPacket воспроизведение FMOD-звуков выполняется через AudioSystem.Instance.PlayAt() в мировых координатах, а визуальные эффекты рендерятся через VFXPool.PooledSlot.
