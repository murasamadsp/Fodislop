# Контрибуция в Fodinae

## Окружение

1. Установите **Unity 6** (`6000.2.10f1`) через Unity Hub.
2. Клонируйте репозиторий. Git-зависимости подтянутся автоматически при открытии проекта в Unity.
3. Основная сцена: `Assets/Scenes/SampleScene.unity`.

## Правила

### Код

- Следуйте стандартам из [`AGENTS.md`](AGENTS.md).
- **Мета-файлы**: Каждый новый файл в `Assets/` должен иметь `.meta`-пару. При удалении/перемещении — обрабатывать оба.
- **GUID**: Не меняйте GUID существующих ассетов. Это ломает ссылки в сценах и префабах.
- **Синглтоны**: Менеджеры используют паттерн `Instance` + `DontDestroyOnLoad`.
- **Координаты**: Помните про инверсию Y между сервером и Unity. Формула: `unityY = WorldHeight - 1 - serverY`.

### Коммиты

- Один коммит — одно логическое изменение.
- Префиксы: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`.
- Пример: `fix: исправить инверсию Y в расчёте теней`

### Pull Requests

1. Создайте ветку от `main`: `git checkout -b feat/my-feature`.
2. Убедитесь, что проект компилируется без ошибок в Unity.
3. Опишите, **что** и **зачем** вы изменили.
4. Если PR затрагивает рендеринг или сетевой слой — приложите скриншот/лог.
